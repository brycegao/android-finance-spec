# Android\-Finance\-Spec 金融级安卓工程规范

## 💡 项目简介

**Android\-Finance\-Spec** 是一套**面向金融行业、可直接落地的企业级 Android \& Kotlin 研发规范**。

区别于通用 Android 规范，本项目**专为理财、银行、信贷、支付、资产类金融 App 定制**，解决金融项目最核心痛点：

- ❌ 浮点精度丢失（资金计算偏差）

- ❌ AI 生成代码混乱、架构不统一

- ❌ 协程崩溃、生命周期泄露

- ❌ 空安全不严谨引发线上闪退

- ❌ 架构混乱、MVVM/MVI 混用无标准

- ❌ 敏感数据日志泄露、不合规

本仓库**原生支持 AI 编码**（Cursor / Claude / Copilot / 通义灵码），是目前 GitHub 稀缺的「**金融专属 \+ 工程标准化 \+ AI 零Bug编码约束**」开源方案。

## ✨ 核心亮点（区别于所有开源规范）

- **💰 金融级数值零误差强制规范（行业独家）**
禁止 Double/Float 存储金额、利率、汇率；
全链路 String 传原始值，计算统一使用 BigDecimal，彻底杜绝资金精度丢失。

- **🤖 原生 AI 编码约束（自然语言级防Bug）**
内置无冲突、无歧义、无逻辑Bug的专业级 AI Prompt，让所有 AI 生成代码**统一架构、统一排版、统一容错、统一金融安全标准**。

- **🔁 MVVM / MVI 双架构自适应**
不强行绑定架构，自动适配项目现有代码风格，避免改造冲突。

- **🛡️ 协程 \& 空安全 金融强校验**
分层异常捕获、禁止裸Launch、生命周期严格绑定、杜绝线上崩溃。

- **🔒 金融合规与隐私安全**
禁止敏感数据日志输出、禁止全局可变状态、禁止内存缓存用户隐私数据。

- **📦 标准化工程目录 \& 分包规范**
统一金融项目分层结构：UI → ViewModel → Domain → Data，业务分包高度规整。

## 📌 规范覆盖范围

- 代码格式、缩进、换行、尾随逗号、行宽

- 统一命名规范（类/函数/常量/扩展/状态流）

- 标准工程分层与业务分包

- MVVM / MVI 双架构自适应规范

- StateFlow / SharedFlow 状态管理标准

- 协程生命周期 \& 异常分级处理

- Kotlin 空安全强制约束

- **金融数值高精度计算规范**

- 隐私安全、日志合规、反崩溃规范

- 类成员固定排序（统一代码审美）
